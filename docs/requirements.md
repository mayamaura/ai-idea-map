# アイデア拡張メモアプリ 要件定義書

**作成日**: 2026-05-27  
**バージョン**: 1.0

---

## 1. プロジェクト概要

### 1.1 アプリ名（仮）
**IdeaMap** - AIと一緒に育てるアイデアマップ

### 1.2 目的
アイデアをノード（円/カード）として配置し、線でつなぐことで思考を可視化・拡張するウェブアプリ。人間の発想とAI（Claude）の提案を組み合わせて、アイデアを深く広く展開できる。

### 1.3 対象ユーザー
- PCおよびスマートフォンで使うソロユーザー
- 自分のClaude APIキーを持つユーザー
- アイデア整理・ブレインストーミングをしたい個人

---

## 2. 機能要件

### 2.1 マインドマップ機能

#### 2.1.1 ノード操作
| 機能 | 説明 |
|------|------|
| ノード追加 | キャンバス上のダブルクリック、または既存ノードから子ノードを追加 |
| ノード編集 | ダブルクリックで詳細パネルを開きタイトル・本文・カテゴリを編集 |
| ノード削除 | 選択後にDeleteキーまたはメニューから削除 |
| ノード移動 | ドラッグ＆ドロップで自由移動 |
| ノードサイズ | テキスト量に応じて自動調整 |
| ノード色 | カテゴリ選択によって自動設定（カテゴリ固有の背景色） |
| ノード本文 | Markdown形式のメモを本文に記載、カードに冒頭2行をプレビュー表示 |
| カテゴリ分類 | ノードをカテゴリ（メインアイデア・問い・アクション等）で分類 |

#### 2.1.2 エッジ（線）操作
| 機能 | 説明 |
|------|------|
| エッジ追加 | ノードのハンドルをドラッグして別ノードに接続 |
| エッジ削除 | エッジを選択してDelete、またはメニューから削除 |
| エッジスタイル | 直線・曲線を選択可能 |
| ラベル | エッジに関係性を示すラベルを付けられる（任意） |

#### 2.1.3 キャンバス操作
| 機能 | 説明 |
|------|------|
| パン（移動） | ドラッグまたは2本指スワイプでキャンバスを移動 |
| ズーム | マウスホイール、ピンチイン/アウト、ズームボタン |
| フィット | 全ノードが画面内に収まるように自動調整 |
| ミニマップ | 右下に全体マップの縮小表示 |

### 2.2 AI拡張機能（Claude連携）

#### 2.2.1 基本動作フロー
1. ユーザーがノードを選択
2. 「AIに拡張を依頼」ボタンをクリック
3. Claude APIにノードのテキストと周辺コンテキストを送信
4. AIが関連アイデアを3〜5個提案
5. ユーザーが採用したい提案を選択
6. 選択した提案が新しいノードとして追加・接続される

#### 2.2.2 AI提案の種類
- **関連アイデア**: 選択ノードに関連する概念・アイデア
- **深掘り**: 選択ノードをより詳細に分解した子アイデア
- **対比・反論**: 反対意見や別の視点
- **応用**: 実際の活用方法・具体的な事例

#### 2.2.3 コンテキスト送信
- 選択したノードのタイトルと本文（body）
- 親ノード・子ノードのタイトル（直接繋がるノード）
- マップ全体のノードタイトルリスト（概要把握のため）
- カテゴリ一覧（AIが提案ノードのカテゴリを自動判定するため）

### 2.3 データ管理

#### 2.3.1 Googleドライブ連携
| 機能 | 説明 |
|------|------|
| 認証 | Google OAuth 2.0でサインイン |
| 保存 | マップをJSONファイルとしてGoogleドライブに保存 |
| 読み込み | Googleドライブ内のマップファイルを一覧表示・読み込み |
| 自動保存 | 変更から数秒後に自動保存（デバウンス処理） |
| 新規作成 | 空のマップを新規作成 |
| 複製 | 既存マップを複製して新しいマップとして保存 |

### 2.3.5 ファイルスキーマ（グループノード対応版）

グループノードの追加に伴い、シリアライズ形式を以下に更新：

```json
{
  "nodes": [
    {
      "id": "uuid",
      "nodeType": "idea",
      "title": "アイデア",
      "x": 100, "y": 200,
      "color": "#ffffff",
      "createdBy": "user",
      "categoryId": "cat-main",
      "parentId": "group-uuid"
    },
    {
      "id": "group-uuid",
      "nodeType": "group",
      "title": "グループ名",
      "x": 50, "y": 100,
      "color": "rgba(147,197,253,0.15)",
      "width": 400,
      "height": 300
    }
  ]
}
```

- `parentId` がある `ideaNode` の `x/y` は **親グループからの相対座標**
- `groupNode` の `width/height` は **リサイズ後のサイズを保存**（リロード後も維持）

### 2.4 グループノード機能（Phase 12）

#### 2.4.1 グループの作成・管理

| 操作 | 説明 |
|------|------|
| グループ作成 | 右クリックメニュー「グループを作成」からキャンバス任意位置に作成 |
| 複数ノードのグループ化 | 2つ以上のアイデアノードを選択した状態で右クリック → 「グループ化（N件）」 |
| グループ解除 | 右クリックメニュー「グループを解除（子ノードは残す）」 |
| グループと子を一括削除 | 右クリックメニュー「グループと子ノードを削除」 |
| グループ名編集 | グループ上でダブルクリック、または右クリックメニュー「ラベルを編集」 |
| グループリサイズ | グループを選択してハンドルをドラッグ（NodeResizerコンポーネント） |

#### 2.4.2 フリーノード → グループへのドラッグイン

1. フリーノードをドラッグ中、グループ枠と重なるとグループが青くハイライトされる
2. グループ上でドロップ（mouseup）した瞬間に確認ダイアログを表示：
   - **「グループに追加」「追加」「キャンセル」**
3. 「追加」確認 → ノードはグループの子になり、位置は相対座標に変換される
4. 「キャンセル」 → ノードはグループの外に押し出される（最小移動距離で隣接位置へ）

**境界判定**：ノードの任意部分がグループ枠内に重なったとき（矩形 AABB 交差判定）

#### 2.4.3 子ノード → グループ外へのドラッグアウト

1. グループ内の子ノードをドラッグし、ノードの**中心点**が親グループ枠の外に出た時点で確認ダイアログを表示：
   - **「グループから外す」「外す」「キャンセル」**
2. 「外す」確認 → ノードはフリーノードになり、絶対座標に変換される
3. 「キャンセル」 → ノードはグループ内に戻される（`clampNodeInsideParent`）

境界判定の基準：ノードの中心（エッジではない）がグループ外に出た時

- これにより、大きめのノードがグループ端付近にいても誤検知しない

#### 2.4.4 グループリサイズ後の境界整合性

- グループをリサイズした後も、ドラッグアウト/ドラッグイン/clamp/シリアライズのすべてが**リサイズ後のサイズを参照する**
- 整列（Auto Layout）によってグループ内ノードが再配置された後も同様
- 技術的背景：`@xyflow/react` の `applyNodeChanges` は `dimensions` change に対して `node.style.width/height` を更新しないため、resize end イベントで `style` を手動同期する

#### 2.4.5 NodeActionBar（AI拡張/詳細/DEL ボタン）の表示位置

- アイデアノードを選択したとき、選択ノードの**直下中央**に NodeActionBar を表示
- グループ内の子ノードを選択した場合も同様に直下中央に表示
- 技術的背景：子ノードの `position` は親グループからの相対座標のため、スクリーン座標変換には `グループ位置 + 子ノード位置 = 絶対座標` を計算した上で `flowToScreenPosition` を使用

#### 2.4.6 グループ内でのノード接続作成

- グループ内の子ノードから「アイデアを作成（接続）」（Tabショートカット or 右クリックメニュー）を実行した場合、新規ノードは**元のノードと同じグループに属した状態**で必ずグループ内に作成される
- グループを自動拡張しない（外部ノードを取り込む副作用を避けるため）
- 配置位置の優先順位：**右 → 下 → 左 → 上** の順に試し、グループ内に完全に収まる最初の方向を採用する
- 4方向すべてがグループ枠をはみ出る場合は、はみ出し量が最小の方向をグループ内にクランプして配置する
- グループ外のフリーノードから接続作成した場合は従来通り右側 +280px に配置する

#### 2.4.7 グループ内子ノードのドラッグ制約

- 子ノードはグループ枠内でのみドラッグ可能（枠外ドラッグ → ダイアログ表示）
- グループ自体をドラッグすると、子ノードは一緒に移動する（React Flow の `parentId` 機構）
- フリーノードをドラッグ中は、子ノードの「グループに追加」ハイライトは表示しない

#### 2.4.7 コンテキストメニュー（グループ内子ノード）

- 子ノード（`parentId` あり）の右クリックメニューには「グループから外す」項目を表示
- グループ右クリックメニューには「グループを解除」「グループと子ノードを削除」を表示

### 2.5 検索 & フィルタリング（Phase 8）

#### 2.4.1 テキスト検索
| 機能 | 説明 |
|------|------|
| 検索バー表示 | Ctrl+F またはツールバー「検索」ボタンでキャンバス上部にオーバーレイ表示 |
| リアルタイム検索 | タイトルと本文を対象にした即時検索 |
| マッチハイライト | マッチしたノードは黄色ボーダーで強調、非マッチはopacity-20でdim |
| マッチ数表示 | 「X / Y件」形式でマッチ数を表示 |
| ノードナビ | ↑↓キーまたはボタンでマッチノード間をfitView移動 |
| 検索結果リスト | ドロップダウンにタイトル＋本文プレビューを一覧表示、クリックでジャンプ |
| 最近使ったノード | 検索クエリ未入力時に最近選択したノード（最大5件）を表示 |

#### 2.4.2 カテゴリフィルター
| 機能 | 説明 |
|------|------|
| フィルターボタン | ツールバーの「フィルター」ボタンからドロップダウンで全カテゴリを表示 |
| OR条件フィルター | 複数カテゴリを選択した場合、いずれかに該当するノードを表示 |
| dim表示 | フィルター対象外のノードをopacity-20でdim（削除はしない） |
| フィルタークリア | 「すべてクリア」ボタンで一括解除 |
| アクティブ表示 | フィルター中はボタンに選択中カテゴリ数をバッジ表示 |

### 2.5 設定機能

| 設定項目 | 説明 |
|----------|------|
| Claude APIキー | ブラウザのlocalStorageに暗号化して保存 |
| AIモデル選択 | claude-sonnet-4-6 / claude-haiku-4-5（コスト重視）の切替 |
| AI提案数 | 3〜7個の範囲で設定 |
| 自動保存間隔 | オン/オフ、間隔設定 |
| テーマ | ライト/ダーク |
| 言語 | 日本語/英語 |

> **Note**: Google OAuth クライアントIDはアプリ側の環境変数（`VITE_GOOGLE_CLIENT_ID`）で管理します。ユーザーが個別に設定する必要はありません。

## 3. 非機能要件

### 3.1 対応環境
| 環境 | 要件 |
|------|------|
| ブラウザ | Chrome 最新版、Safari 最新版、Edge 最新版 |
| PC | Windows / macOS / Linux（上記ブラウザ経由） |
| スマホ | iOS 16以上、Android 12以上 |
| 画面サイズ | 320px〜4Kまで対応（レスポンシブ） |

### 3.2 パフォーマンス
- ノード数100個以下でスムーズな操作（60fps）
- 初回ロード3秒以内
- AI応答は最大30秒のタイムアウト表示

### 3.3 セキュリティ
- Claude APIキーはlocalStorageに保存（サーバーに送信しない）
- Google OAuth トークンは標準的なセキュリティプラクティスに従う
- APIキーを含むデータは外部サービスに送信しない

### 3.4 オフライン対応
- Googleドライブ未接続時はlocalStorageに一時保存
- 再接続時に同期

## 4. UI/UX要件

### 4.1 画面構成
```
+------------------------------------------+
| [タイトル]  [保存中...] [設定] [Googleログイン] |  ← ヘッダー
+------------------------------------------+
|                                          |
|           メインキャンバス               |
|      （マインドマップ表示領域）           |
|                                  [ミニマップ] |
+------------------------------------------+
| [+ノード] [自動整列] [ズームイン][ズームアウト][フィット] |  ← ツールバー
+------------------------------------------+
```

### 4.2 ノード選択時のパネル
```
+---------------------------+
|  選択中: "アイデアA"      |
|  [編集] [削除] [色変更]   |
|  ─────────────────────── |
|  [🤖 AIに拡張を依頼]      |
+---------------------------+
```

### 4.3 AI提案パネル
```
+----------------------------------+
|  AIからの提案 (4件)         [×]  |
|  ─────────────────────────────  |
|  [✓] 関連アイデア1               |
|  [✓] 関連アイデア2               |
|  [ ] 関連アイデア3               |
|  [✓] 関連アイデア4               |
|  ─────────────────────────────  |
|  [選択した提案を追加] [再生成]    |
+----------------------------------+
```

### 4.4 スマホ対応
- タッチ操作対応（タップ、ドラッグ、ピンチ）
- ツールバーはボトムナビゲーションに変更
- ノード操作はロングプレスでコンテキストメニュー表示

## 5. 技術的制約

- **フロントエンドのみ**（バックエンドサーバー不要）
- Claude APIは直接ブラウザから呼び出し（CORS対応のためプロキシ不要）
- Googleドライブ APIはクライアントサイドのみで操作
- 静的ホスティング（GitHub Pages、Vercel、Netlifyなど）で公開可能
